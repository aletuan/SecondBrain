# Category taxonomy for captures — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Auto-assign multi-label `categories` (fixed taxonomy) on ingest via LLM; validate and persist to `*.note.md` frontmatter; allow manual edit in reader-web. Re-ingest overwrites categories.

**Architecture:** Load taxonomy from `config/categories.yaml` with fallback to `categories.example.yaml` (same pattern as `readRoutingYamlSync`). New `extractCategories` LLM call returns JSON array of ids; filter to allowed ids; `setCategoriesInNoteFrontmatter` replaces the key. Wire into `runIngest` beside existing `extractTags`/`addTagsToNoteFrontmatter`. Reader-web loads taxonomy server-side and renders multi-select saving to note.

**Tech Stack:** TypeScript (Node 20+), OpenAI chat completions (existing client), YAML (use existing project yaml dep if any — check `package.json`), Vitest.

**Spec:** [`docs/superpowers/specs/2026-03-29-category-design.md`](../specs/2026-03-29-category-design.md)
